知识总结

1、作用域链

每一个函数都有一个作用域链，如，创建了一个函数，函数里又包含了一个函数，这样就形成了作用域链
特点：在组件作用域的范围去查找，如果找不到，就会沿着作用域链向上查找

2、MVC和MVVM的区别

1)MVVM定义

MVVM是Model-View-ViewModel的简写。即模型-视图-视图模型
模型：指的是后端传递的数据
视图：指的是看到的页面
视图模型：MVVM的核心，它是连接View和Model的桥梁（数据绑定和dom事件监听）

2)MVC定义

MVC是Model-View-Controller的简写。即模型-视图-控制器
M：数据 将js的ajax当做Model，就是数据层
V：看到的页面，如文字、数字、日期等，方便控制器控制。可以将html看成是View
C：js功能控制 将js看成是Controller控制器

3)MVC和MVVM的区别

MVVM：实现了View和Model的自动同步，就是Model的属性改变，不需要手动操作DOM，来改变View的显示，而是改变后，对应的View层会自动改变，例，Vue

3、js模块化

CMD推崇就近依赖，只有在用到某个模块的时候再去require 依赖于seajs 异步加载

AMD推崇依赖前置，在定义模块的时候就要声明其依赖的模块 依赖于requirejs 异步加载

commonJS运行在服务端 同步加载

ES6是ES2015的简称，一般通过export来暴露模块，import来导入模块（国际标准）

4、请用js去除字符串空格？

1)使用replace正则匹配的方法

去除所有空格：str=str.replace(/\s*/g,"");
去除两头空格：str=str.replace(/^\s*|\s*$/g,"");
去除左空格：str=str.replace(/^\s*/,"");
去除右空格：str=str.replace(/(\s*$)/g,"");
var str="23 23";
var str2=str.replace(/\s*/g,"");
console.log(str2);//2323

2)使用str.trim()方法

str.trim()局限性，无法去除中间的空格
var str="23 23";
var str2=str.trim();
console.log(str2);//23 23
同理，str.trimLeft(),str.trimRight()分别去除字符串的左右空格

3)使用jquery，$.trim(str)方法

$.trim(str)局限性，无法去除中间的空格
var str="23 23";
var str2=str.trim();
console.log(str2);//23 23

6、常用的字符串操作方法？

concat()-将两个或多个字符拼接，返回一个新的字符串
indexOf()-返回字符串中一个字串第一次出现的索引位置，不存在返回-1
lastIndexOf()-返回字符串中一个字串最后一次出现的索引位置，不存在返回-1
charAt()-返回指定位置的字符串
match()-检查一个字符串是否匹配一个正则表达式
substr()函数-返回从string的startPos位置，长度为length的字符串
substring()-返回字符串的一个字串，参数是起始位置和结束位置
slice()-提取字符串的一部分，返回一个新的字符串
replace()-用来查找匹配一个正则表达式的字符串，然后使用新字符串代替匹配的字符串
search()-执行一个正则表达式匹配查找，如果查找成功，返回字符串中匹配的索引值，否则返回-1
split()-将一个字符串拆成一个字串数组
length-返回字符串长度
toLowerCase()-将整个字符串转成小写字母
toUpperCase()-将整个字符串转成大写字母
